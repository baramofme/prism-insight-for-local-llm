# Google Finance (베타) 레이아웃 반응형 스펙

> 대상: **finance.google.com/finance/beta** (종목/홈)
> 조사 방법: DevTools 콘솔 클릭-픽 인스펙터로 뷰포트별 각 영역의 width/display/flex/padding/margin/position 실측
> 측정 뷰포트: 678 · 724 · 780 · 874 · 1074 · 1380 · 1680 · 2170 px
> 목적: alpha(Next.js) 구글 파이낸스 클론의 레이아웃 기준 (Single Source of Truth)

## 1. 브레이크포인트 정의

| 구간 | 좌측 네비 | 우측 조사 패널 | 전체 컨테이너 |
|---|---|---|---|
| **< ~760px** | 숨김 | 숨김 | 풀폭, 좌우 16px 패딩 |
| **~760 ~ 1380px** | **80px 고정 레일** (호버 시 320px 오버레이) | ~1000px↑부터 등장 | 풀폭 |
| **1380 ~ 1480px** | **300px 자동 펼침** (콘텐츠 밀림) | 유동 | 풀폭 |
| **≥ 1480px** | **320px 자동 펼침** | 유동 | 풀폭 |
| **≥ 1820px** | 320px (상한) | 유동 | **max-width 1820px + 가운데 정렬** |

> 핵심 임계값: **760**(네비 등장) · **~1000**(우측 패널 등장) · **1380**(좌측 자동 펼침 300) · **1480**(320) · **1820**(전체 캡·센터링)

## 2. 영역 식별자 (난독화 클래스 → 기능)

재조사 시 앵커로 사용. (GF가 동적 렌더링하므로 클래스명은 변할 수 있음)

| class | 기능 |
|---|---|
| `YXDqL` | 헤더 (Finance 로고 / 검색 / 시장 네비) — `position: absolute` |
| `t4pbz` / `d0WT0b` | **좌측 네비(목록) 레일** — `position: fixed` |
| `UHCxXe` | 좌측 네비 내부 항목(포트폴리오 등) — 펼침 시 콘텐츠 폭 측정 기준 |
| `Y8k45b` | **메인 콘텐츠** (홈/시장 본문) |
| `OWRwWb` | **우측 조사(Research) 패널** — `flex-grow: 1` |
| `DrZEDd` | 푸터 (AI 면책) — `max-width: 1820px` |

## 3. 좌측 네비 (Left Nav / 목록)

**메커니즘**: `position: fixed` 고정 레일. 폭은 브레이크포인트로 결정(고정폭). 펼침 시 메인 콘텐츠를 `margin-left`로 밀어냄. 단 760~1380 구간의 **호버 펼침은 오버레이**(콘텐츠 안 밂).

| 뷰포트 | 레일 폭 | 콘텐츠 margin-left | 내부 항목 콘텐츠 (좌우 24px 패딩) | 동작 |
|---|---|---|---|---|
| <760 | 숨김 (display:none) | 0 | — | — |
| 760~1380 | **80px** | 80 | — | 아이콘 레일 / **호버 → 320px 오버레이** |
| 1380~1480 | **300px** | 300 (밀림) | **252px** (300−48) | 자동 펼침 |
| ≥1480 | **320px** | 320 (밀림) | **272px** (320−48) | 자동 펼침 |

- 펼침 폭은 ≥1480에서 **320px 상한** (더 넓어지지 않음).
- 내부 패딩 = **좌우 24px** 고정 → 콘텐츠 = 레일폭 − 48.

---

## 4. 메인 콘텐츠 (Main Content)

**메커니즘**: **유동폭**(max-width 없음). `margin-left` = 좌측 레일 폭, 좌우 **padding 24px**. 우측 조사 패널이 있으면 그만큼 좁아짐.

| 뷰포트 | 콘텐츠 width | margin-left | padding L/R | 비고 |
|---|---|---|---|---|
| 780 | 700 | 80 | 24 | 우측 패널 없음 (700 = 780−80) |
| 874 | 794 | 80 | 24 | 우측 패널 없음 |
| 1074 | 650 | 80 | 24 | 우측 패널 등장(~314) |
| 1380 | 736 | 300 | 24 | 좌 300 + 본문 736 + 우 314 |
| 1680 | 840 | 320 | 24 | 좌 320 + 본문 840 + 우 490 |
| 2170 | 840 | 495 | 24 | 495 = 175(센터링) + 320(레일); 전체 1820 캡 |

## 5. 우측 조사 패널 (Research Panel)

**메커니즘**: **유동(`flex-grow: 1`)**. ~1000px↑부터 등장. 뷰포트가 커질수록 콘텐츠와 함께 성장.

| 뷰포트 | 패널 width |
|---|---|
| ≤874 | 없음 |
| 1074 | ~314 |
| 1380 | 314 |
| 1680 | 490 |
| 2170 | 628 |

- 조사 패널도 호버 시 펼쳐지는 레일 형태(`t4pbz "조사…"`)가 있으며 폭은 본문 우측 영역과 함께 유동.

---

## 6. 전체 컨테이너 / 헤더 / 푸터

- **전체 컨테이너**: **max-width 1820px**. 뷰포트 > 1820이면 **가운데 정렬**(좌우 균등 여백). 2170px에서 본문 margin-left 495 = (2170−1820)/2 + 320.
- **헤더 `YXDqL`**: `position: absolute`, 폭은 1820 캡까지 풀폭.
- **푸터 `DrZEDd`**: `max-width 1820px`, padding 8px, 가운데 정렬.
- 콘텐츠 좌우 거터: **24px** (펼침 구간), 모바일 16px.
